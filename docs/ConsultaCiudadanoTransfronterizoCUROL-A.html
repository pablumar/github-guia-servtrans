Transfronterizo - ROL A Consulta de un ciudadano desde servicio transfronterizo

Diagrama de Secuencia:

​

Diagrama de Aplicación:

​

Descripción del Escenario:

El escenario propuesto es el siguiente: desde un servicio transfronterizo se inicia una consulta de datos demográficos de un paciente utilizando el mensaje "Mobile Patient Demographics Query ITI-78". Esta consulta se envía al "Patient Demographics Supplier", que es el sistema TSI en este contexto. El "Patient Demographics Supplier" procesa la consulta y responde con los datos demográficos solicitados utilizando el mensaje "Mobile Patient Demographics Query Response". Esta respuesta incluye únicamente los datos mínimos definidos: identificador, en este caso el CIP-SNS, nombre y apellidos, sexo, fecha de nacimiento y estado.

En resumen, el Flujo de Proceso Básico de la transacción ITI-78 en el perfil PDQm implica la solicitud y respuesta de datos demográficos de pacientes entre el consumidor y el proveedor de datos demográficos, con el sistema TSI desempeñando el papel de proveedor de datos demográficos en este escenario. Este flujo facilita la interoperabilidad y el intercambio de información demográfica de pacientes entre sistemas de salud y aplicaciones móviles, mejorando así la accesibilidad y la coordinación de la atención médica.

El resultado de esta consulta proporcionará al servicio transfronterizo la información mínima del paciente, nombre y apellidos, sexo, fecha de nacimiento e identificadores que tiene asignados el paciente, minimizando con ello el envío de información.

Operación involucrada: /Patient

Ejemplo:

  • Solicitud: Desde un servicio transfronterizo se realiza una consulta de un paciente con un identificador con un OID asignado 2.16.724.4.40, que corresponde a CIP-SNS, y un número de documento: BBBBBBBBBB009933 y únicamente se recuperarán los datos mínimos.

GET /Patient?identifier=2.16.724.4.40|BBBBBBBBBB009933 &_elements=identifier,active,name,birthDate,gender

Otro tipo de búsqueda que se puede realizar es por CIP+CITE, para ello el formato sería el siguiente:

GET /Patient?identifier=2.16.724.4.42.803416|GRSN650112917048&_elements=identifier,active,name,birthDate,gender

Donde el OID 2.16.724.4.42 indica que es un CITE lo que se indica en la última parte del OID: 803416.
